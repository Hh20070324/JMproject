# JM 漫画下载器 v2.5.1 用户指南

by TYZQ

## 发行版快速开始

1. 下载 `JM-Downloader-v2.5.1-Windows-x64.zip` 及同名 `.sha256` 文件。
2. 在 PowerShell 运行 `Get-FileHash .\JM-Downloader-v2.5.1-Windows-x64.zip -Algorithm SHA256`，确认结果与 `.sha256` 文件一致。
3. 解压 ZIP，保持文件夹结构完整，双击 `JM-Downloader.exe`。
4. 进入「搜索与下载」，切换到「下载任务」标签页。
5. 输入纯数字 JM 号，按回车或点击「开始下载」。

发行版已经包含 Python 和运行依赖，不需要安装开发环境。请整体移动解压后的 `JM-Downloader` 文件夹，不要只移动 EXE，也不要遗漏 `_internal`。

发行目录中有两个可执行文件：

- `JM-Downloader.exe`：日常使用的正式版。
- `JM-Downloader-Debug.exe`：显示终端日志的调试版，启动或下载异常时使用。

两个 EXE 使用同一份 `settings.json`、图片目录和 PDF 目录。

## 下载漫画

1. 在左侧进入「搜索与下载」。
2. 点击页面上方的「下载任务」标签。
3. 在任务输入框中填写 JM 号，例如 `1449491`。
4. 按回车或点击「开始下载」。
5. 程序读取章节目录后，单章漫画直接加入队列；多章漫画打开「章节选择」。默认只勾选第一章，可以非连续选择或使用「全选」，至少选择一章才能确认。
6. 需要批量下载时，继续输入其他 JM 号即可加入队列。
7. 等待任务状态变为「已完成」。

任务会显示漫画标题、当前阶段、实时进度和已下载的首张图片。任务行提供暂停、继续和取消按钮；失败任务可以继续下载。取消时可以只移除任务并保留文件，也可以删除该 JM 在任务目录中的全部章节图片与 PDF。删除选项会二次确认，并包含本次任务开始前已经存在的章节文件。

程序会验证已有和新下载图片，损坏或缺失的图片会重新下载。只有全部图片完整时才会生成 PDF。任务完成后会保留约 5 秒，随后从任务页自动移除；图片和 PDF 继续由「本地漫画库」管理。

未完成任务会写入程序目录的 `tasks.json`，其中包括创建任务时实际选择的固定章节 ID。关闭窗口、程序异常结束或网络失败后，下次启动会恢复暂停或失败任务，但不会自动联网。点击继续或失败后重试时，程序只处理原选章节并复用有效图片，不会自动加入远端后来新增的章节。

默认最多同时下载 2 个任务，每个任务默认并发下载 16 张图片。可以在「设置」中分别调整为 1 至 8 个任务、1 至 64 张图片，保存后重启程序生效。

### 搜索漫画

「搜索结果」标签页提供两种查询入口：

1. 在左侧选择「综合」「作者」或「标签」，输入关键词后按回车或点击搜索按钮。
2. 需要按编号查找时，在右侧输入纯数字或带 `JM` 前缀的编号并提交。
3. 查看结果卡片中的标题、作者、JM 号、标签和封面。上游没有返回作者或标签时，程序不会自行猜测。
4. 普通结果初始显示「下载」；点击后才读取章节，期间按钮显示「读取章节…」。单章结果显示「下载整本」并直接入队，多章结果显示「章节选择」。
5. 卡片显示「查看任务」时，点击它会切换到下载任务列表，不会重复添加任务。

普通搜索支持上一页和下一页，最多展示前 1000 条结果。网络暂时不可用或上游响应异常时，可以在错误页面重试，不需要重启程序。切换到下载任务再返回时，当前搜索结果会保留。

当前版本不保存搜索历史，也不提供独立漫画详情页。

## 账号与我的收藏

1. 从左侧进入「我的收藏」，输入账号和密码后登录。密码只在本次登录请求期间使用，不会写入磁盘。
2. 点击「同步」读取默认收藏夹、自建收藏夹和全部分页。同步是手动操作，启动程序或进入页面不会自动联网。
3. 同步成功后可以选择收藏夹并翻页。每页最多显示 20 张卡片；收藏卡片与搜索结果使用相同的单章下载或多章选择流程。
4. 同步失败、断网或点击停止时，程序不会保存半份结果，而是继续显示上次完整同步的内容。
5. 会话过期后仍可离线查看旧缓存；重新输入密码登录并手动同步即可更新。
6. 点击「退出登录」并确认后，程序会删除本地账号会话和收藏缓存。

登录后，搜索结果卡片左下角的收藏按钮会从灰色变为可用。点击后只会添加到默认收藏；成功时卡片立即显示已收藏，并提示进入「我的收藏」手动同步。程序不会自动切换页面或自动同步。未登录、登录中、会话过期或账号数据不可读时，收藏按钮保持灰色禁用。

当前版本不支持取消收藏、选择目标收藏文件夹、移动收藏或管理远端文件夹。写入遇到断网或无法确认的响应时不会自动重试，请手动同步确认服务器最终状态。登录后会在程序目录按需创建 `account.dat`，首次成功同步后创建 `favorites.dat`；两者均由 Windows DPAPI CurrentUser 加密。它们可以随程序目录在同一台电脑、同一 Windows 用户下移动，但通常不能在其他电脑或其他 Windows 用户下解密。遇到本地数据不可读提示时，可以清除这两个文件并重新登录；程序不会自动删除无法解密的原文件。

## 本地漫画库

「本地漫画库」会扫描当前设置的图片和 PDF 目录。进入该页后可以：

- 按 JM 号搜索。
- 查看全部项目，或筛选有图片、有 PDF 的项目。
- 打开图片目录。
- 使用系统默认程序查看 PDF。
- 从已有图片生成或重新生成 PDF。
- 分别删除图片、PDF，或删除全部本地文件。

如果在程序外增删了文件，点击工具栏中的刷新按钮重新扫描。正在下载或处理本地文件的项目会暂时禁止删除和重新生成。

本地漫画库仍按 JM 号整本聚合，不展开章节，也不支持单章删除。分批下载时，图片会累积在同一 JM 目录下的章节子目录；任务完成后重新生成的单个 `JM号.pdf` 会包含该目录中当时存在的全部有效章节图片，而不只包含本次勾选的章节。

## 设置

左侧「设置」页支持以下项目：

| 设置 | 范围或选项 | 说明 |
|------|------------|------|
| 漫画图片目录 | 相对或绝对路径 | 默认为 `Pictures/` |
| PDF 目录 | 相对或绝对路径 | 默认为 `PDFs/` |
| 同时下载 | 1 至 8 个任务 | 默认为 2 |
| 图片并发 | 1 至 64 张图片 | 默认为 16 |
| 日志级别 | 调试、信息、警告、错误 | 默认为信息 |
| 启动页面 | 搜索与下载、本地漫画库、我的收藏、设置 | 下次启动生效 |
| 窗口尺寸 | 宽度与高度 | 保存并在退出时记录 |
| 主题 | 明亮、黑暗 | 保存后立即更新 |

设置保存在程序目录的 `settings.json`。程序目录内的路径会保存为相对路径，因此整体移动程序文件夹后仍然可用；程序目录外的路径会保存为绝对路径。已经创建的未完成任务会继续使用创建时的图片和 PDF 目录，不会因后来修改设置而拆分文件。

主题和窗口尺寸可在当前会话更新。输出目录、下载并发、日志级别和启动页面等设置请在保存后重启程序，以确保完整生效。

## 文件输出

| 默认路径 | 内容 | 结构示例 |
|----------|------|----------|
| `Pictures/` | 下载的原始图片 | `Pictures/JM号/章节名/00001.jpg` |
| `PDFs/` | 自动生成的 PDF | `PDFs/JM号.pdf` |
| `logs/` | 程序日志 | `logs/app.log` |
| `tasks.json` | 未完成和失败任务 | 程序自动维护，不要手动编辑 |
| `account.dat` | 加密账号会话 | 登录后按需创建；密码不保存在其中 |
| `favorites.dat` | 加密收藏缓存 | 完整同步成功后按需创建 |

图片和 PDF 目录可以在设置页修改，不一定位于程序目录。

`option.yml` 保留下载器的底层选项，例如图片后缀。图片并发请使用设置页，不要通过 `option.yml` 调整。请求超时和请求级重试由程序显式设置。

下载中的单张图片会先写入带 `.jm-part-` 标记的临时文件，完整校验后才替换最终图片。陈旧临时文件会在继续任务时清理，也不会进入预览、本地库或 PDF。

## 源码版（开发者）

源码项目要求 64 位 Windows 和 Python 3.10 至 3.14。双击 `start.bat` 会自动调用 `scripts/setup.ps1` 创建 `.venv`、安装依赖，然后从正式入口 `desktop.py` 启动。

手动命令：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\setup.ps1
.\.venv\Scripts\python.exe desktop.py
```

## 常见问题

### 正式版双击后没有窗口？

运行 `JM-Downloader-Debug.exe` 查看终端错误，并检查 `logs/app.log`。程序目录和设置的下载目录必须可写。

### 提示依赖缺失？

源码版重新运行 `scripts/setup.ps1`。发行版不应手动安装依赖；请确认 `_internal` 完整，必要时重新解压发行包。

### 下载报错 `Restricted Access` 或一直等待？

当前网络出口可能受到目标站点限制。确认网络和代理设置后重试。

### 设置损坏或配置的目录不可用？

先退出程序。可以修正 `settings.json`，或删除它以恢复默认设置。损坏的设置通常会自动备份为 `settings.json.corrupt-*`。

### 关闭窗口后任务还在吗？

在。确认关闭后，程序停止调度新图片，并等待已经发出的网络请求返回。请求不能保证瞬间中断，但已完整取得的图片会安全保存。未完成任务在下次启动时以暂停状态恢复，只有点击继续后才会联网。

### 暂停、取消和失败后继续分别会发生什么？

- 暂停：保留任务、固定章节选择、图片和已有 PDF，稍后可继续原选章节。
- 取消并保留文件：移除任务记录，不删除图片和 PDF。
- 取消并删除文件：二次确认后等待下载线程停止，再删除该 JM 的全部章节图片和 PDF，包括任务开始前已有的章节。
- 失败后继续：保留有效图片，只重新下载缺失或损坏内容。

暂停和取消采用协作式停止：已经发出的网络请求允许返回，不代表立即切断当前连接。

### 恢复任务后会自动下载远端新增章节吗？

不会。任务保存创建时实际勾选的章节 ID，暂停、重启和失败后继续都沿用这份固定选择。需要下载后来新增的章节时，请在旧任务结束或移除后重新发起下载并选择章节。

### 修改下载目录会影响恢复任务吗？

不会。恢复任务始终使用创建时的目录，避免同一本漫画被拆分到新旧位置。当前本地漫画库只扫描设置中的当前目录，因此旧目录任务完成后可先从任务行打开其原目录。

### 复制到其他电脑能用吗？

可以。整体复制解压后的程序文件夹，不要遗漏 `_internal`。相对于程序目录的图片和 PDF 路径会随目录移动；设置为外部绝对路径的内容需要单独复制。`account.dat` 和 `favorites.dat` 绑定原 Windows 用户，换电脑后通常需要清除本地账号数据并重新登录。

### 登录信息保存在哪里？

程序只把加密会话保存在 EXE 同级的 `account.dat`，收藏缓存在同级的 `favorites.dat`；不会写入 AppData、注册表或 Windows 凭据管理器。密码本身不会保存。普通搜索和下载不会使用账号 Cookie。

## 项目结构

```text
JM-Download&wrap_program/
├── desktop.py               # Qt 正式入口
├── jm_downloader/
│   ├── qt/                  # 原生桌面界面
│   ├── tasks.py             # 下载任务
│   ├── task_store.py         # 任务恢复记录
│   ├── account.py           # 账号与会话
│   ├── favorites.py         # 收藏同步与缓存
│   ├── protected_store.py   # Windows DPAPI 加密存储
│   ├── downloader.py        # 下载适配
│   ├── library.py           # 本地漫画库
│   ├── pdf.py               # PDF 生成
│   └── settings.py          # 设置与便携路径
├── option.yml               # 下载器底层配置
├── requirements.txt         # 源码运行依赖
├── LICENSES/                # 第三方许可证文本
├── QT_SOURCE_AND_RELINKING.md
├── QT_THIRD_PARTY_NOTICES.txt
├── scripts/
│   ├── setup.ps1            # 源码环境配置
│   └── build.ps1            # 发行包构建
├── start.bat                # 源码快捷启动
├── Pictures/                # 默认图片目录
├── PDFs/                    # 默认 PDF 目录
├── settings.json            # 运行时设置
├── tasks.json               # 任务恢复记录，按需创建
├── account.dat              # 加密账号会话，按需创建
└── favorites.dat            # 加密收藏缓存，按需创建
```

## 构建发行包

准备好项目虚拟环境后运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\build.ps1
```

脚本会运行测试、构建正式版和调试版，生成 `release/JM-Downloader-v2.5.1-Windows-x64.zip` 及同名 `.sha256` 校验文件。它不会覆盖 v2.1.0、v2.2.0、v2.3.0、v2.4.0 和 v2.5.0 历史发行包，也会拒绝把设置、任务记录、账号会话、收藏缓存、日志、下载内容和临时文件打入发行 ZIP。
