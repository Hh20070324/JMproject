# JM 漫画下载器 v2.4.0 用户指南

by TYZQ

## 发行版快速开始

1. 下载 `JM-Downloader-v2.4.0-Windows-x64.zip` 及同名 `.sha256` 文件。
2. 在 PowerShell 运行 `Get-FileHash .\JM-Downloader-v2.4.0-Windows-x64.zip -Algorithm SHA256`，确认结果与 `.sha256` 文件一致。
3. 解压 ZIP，保持文件夹结构完整，双击 `JM-Downloader.exe`。
4. 进入「搜索与下载」，切换到「下载任务」标签页。
5. 输入纯数字 JM 号，按回车或点击「开始下载」。

发行版已经包含 Python 和运行依赖，不需要安装开发环境。请整体移动解压后的 `JM-Downloader` 文件夹，不要只移动 EXE，也不要遗漏 `_internal`。

发行目录中有两个可执行文件：

- `JM-Downloader.exe`：日常使用的正式版。
- `JM-Downloader-Debug.exe`：显示终端日志的调试版，启动或下载异常时使用。

两个 EXE 使用同一份 `settings.json`、图片目录和 PDF 目录。

## 下载漫画

1. 在左侧进入「搜索与下载」。
2. 点击页面上方的「下载任务」标签。
3. 在任务输入框中填写 JM 号，例如 `1449491`。
4. 按回车或点击「开始下载」。
5. 需要批量下载时，继续输入其他 JM 号即可加入队列。
6. 等待任务状态变为「已完成」。

任务会显示漫画标题、当前阶段、实时进度和已下载的首张图片。任务行提供暂停、继续和取消按钮；失败任务可以继续下载。取消时可以只移除任务并保留文件，也可以同时删除已经下载的图片与 PDF。

程序会验证已有和新下载图片，损坏或缺失的图片会重新下载。只有全部图片完整时才会生成 PDF。任务完成后会保留约 5 秒，随后从任务页自动移除；图片和 PDF 继续由「本地漫画库」管理。

未完成任务会写入程序目录的 `tasks.json`。关闭窗口、程序异常结束或网络失败后，下次启动会恢复暂停或失败任务，但不会自动联网。点击继续后，程序复用有效图片并只补齐缺失内容。

默认最多同时下载 2 个任务，每个任务默认并发下载 16 张图片。可以在「设置」中分别调整为 1 至 8 个任务、1 至 64 张图片，保存后重启程序生效。

### 搜索漫画

「搜索结果」标签页提供两种查询入口：

1. 在左侧选择「综合」「作者」或「标签」，输入关键词后按回车或点击搜索按钮。
2. 需要按编号查找时，在右侧输入纯数字或带 `JM` 前缀的编号并提交。
3. 查看结果卡片中的标题、作者、JM 号、标签和封面。上游没有返回作者或标签时，程序不会自行猜测。
4. 点击「下载整本」将漫画加入下载队列；搜索本身不会开始下载。
5. 卡片显示「查看任务」时，点击它会切换到下载任务列表，不会重复添加任务。

普通搜索支持上一页和下一页，最多展示前 1000 条结果。网络暂时不可用或上游响应异常时，可以在错误页面重试，不需要重启程序。切换到下载任务再返回时，当前搜索结果会保留。

当前版本不保存搜索历史，不提供漫画详情页或章节选择。所有下载仍以整本漫画为单位。

## 本地漫画库

「本地漫画库」会扫描当前设置的图片和 PDF 目录。进入该页后可以：

- 按 JM 号搜索。
- 查看全部项目，或筛选有图片、有 PDF 的项目。
- 打开图片目录。
- 使用系统默认程序查看 PDF。
- 从已有图片生成或重新生成 PDF。
- 分别删除图片、PDF，或删除全部本地文件。

如果在程序外增删了文件，点击工具栏中的刷新按钮重新扫描。正在下载或处理本地文件的项目会暂时禁止删除和重新生成。

## 设置

左侧「设置」页支持以下项目：

| 设置 | 范围或选项 | 说明 |
|------|------------|------|
| 漫画图片目录 | 相对或绝对路径 | 默认为 `Pictures/` |
| PDF 目录 | 相对或绝对路径 | 默认为 `PDFs/` |
| 同时下载 | 1 至 8 个任务 | 默认为 2 |
| 图片并发 | 1 至 64 张图片 | 默认为 16 |
| 日志级别 | 调试、信息、警告、错误 | 默认为信息 |
| 启动页面 | 搜索与下载、本地漫画库、设置 | 下次启动生效 |
| 窗口尺寸 | 宽度与高度 | 保存并在退出时记录 |
| 主题 | 明亮、黑暗 | 保存后立即更新 |

设置保存在程序目录的 `settings.json`。程序目录内的路径会保存为相对路径，因此整体移动程序文件夹后仍然可用；程序目录外的路径会保存为绝对路径。已经创建的未完成任务会继续使用创建时的图片和 PDF 目录，不会因后来修改设置而拆分文件。

主题和窗口尺寸可在当前会话更新。输出目录、下载并发、日志级别和启动页面等设置请在保存后重启程序，以确保完整生效。

## 文件输出

| 默认路径 | 内容 | 结构示例 |
|----------|------|----------|
| `Pictures/` | 下载的原始图片 | `Pictures/JM号/章节名/00001.jpg` |
| `PDFs/` | 自动生成的 PDF | `PDFs/JM号.pdf` |
| `logs/` | 程序日志 | `logs/app.log` |
| `tasks.json` | 未完成和失败任务 | 程序自动维护，不要手动编辑 |

图片和 PDF 目录可以在设置页修改，不一定位于程序目录。

`option.yml` 保留下载器的底层选项，例如图片后缀。图片并发请使用设置页，不要通过 `option.yml` 调整。请求超时和请求级重试由程序显式设置。

下载中的单张图片会先写入带 `.jm-part-` 标记的临时文件，完整校验后才替换最终图片。陈旧临时文件会在继续任务时清理，也不会进入预览、本地库或 PDF。

## 源码版（开发者）

源码项目要求 64 位 Windows 和 Python 3.10 至 3.14。双击 `start.bat` 会自动调用 `scripts/setup.ps1` 创建 `.venv`、安装依赖，然后从正式入口 `desktop.py` 启动。

手动命令：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\setup.ps1
.\.venv\Scripts\python.exe desktop.py
```

## 常见问题

### 正式版双击后没有窗口？

运行 `JM-Downloader-Debug.exe` 查看终端错误，并检查 `logs/app.log`。程序目录和设置的下载目录必须可写。

### 提示依赖缺失？

源码版重新运行 `scripts/setup.ps1`。发行版不应手动安装依赖；请确认 `_internal` 完整，必要时重新解压发行包。

### 下载报错 `Restricted Access` 或一直等待？

当前网络出口可能受到目标站点限制。确认网络和代理设置后重试。

### 设置损坏或配置的目录不可用？

先退出程序。可以修正 `settings.json`，或删除它以恢复默认设置。损坏的设置通常会自动备份为 `settings.json.corrupt-*`。

### 关闭窗口后任务还在吗？

在。确认关闭后，程序停止调度新图片，并等待已经发出的网络请求返回。请求不能保证瞬间中断，但已完整取得的图片会安全保存。未完成任务在下次启动时以暂停状态恢复，只有点击继续后才会联网。

### 暂停、取消和失败后继续分别会发生什么？

- 暂停：保留任务、图片和已有 PDF，稍后可继续。
- 取消并保留文件：移除任务记录，不删除图片和 PDF。
- 取消并删除文件：等待下载线程停止，然后安全删除该任务绑定目录中的内容。
- 失败后继续：保留有效图片，只重新下载缺失或损坏内容。

暂停和取消采用协作式停止：已经发出的网络请求允许返回，不代表立即切断当前连接。

### 修改下载目录会影响恢复任务吗？

不会。恢复任务始终使用创建时的目录，避免同一本漫画被拆分到新旧位置。当前本地漫画库只扫描设置中的当前目录，因此旧目录任务完成后可先从任务行打开其原目录。

### 复制到其他电脑能用吗？

可以。整体复制解压后的程序文件夹，不要遗漏 `_internal`。相对于程序目录的图片和 PDF 路径会随目录移动；设置为外部绝对路径的内容需要单独复制。

## 项目结构

```text
JM-Download&wrap_program/
├── desktop.py               # Qt 正式入口
├── jm_downloader/
│   ├── qt/                  # 原生桌面界面
│   ├── tasks.py             # 下载任务
│   ├── task_store.py         # 任务恢复记录
│   ├── downloader.py        # 下载适配
│   ├── library.py           # 本地漫画库
│   ├── pdf.py               # PDF 生成
│   └── settings.py          # 设置与便携路径
├── option.yml               # 下载器底层配置
├── requirements.txt         # 源码运行依赖
├── LICENSES/                # 第三方许可证文本
├── QT_SOURCE_AND_RELINKING.md
├── QT_THIRD_PARTY_NOTICES.txt
├── scripts/
│   ├── setup.ps1            # 源码环境配置
│   └── build.ps1            # 发行包构建
├── start.bat                # 源码快捷启动
├── Pictures/                # 默认图片目录
├── PDFs/                    # 默认 PDF 目录
├── settings.json            # 运行时设置
└── tasks.json               # 任务恢复记录，按需创建
```

## 构建发行包

准备好项目虚拟环境后运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\build.ps1
```

脚本会运行测试、构建正式版和调试版，生成 `release/JM-Downloader-v2.4.0-Windows-x64.zip` 及同名 `.sha256` 校验文件。它不会覆盖 v2.1.0、v2.2.0 和 v2.3.0 历史发行包，也会拒绝把设置、任务记录、日志、下载内容和临时图片打入发行 ZIP。
