# 重构回归基线

第一阶段只建立行为基线，不改变现有下载流程。

## 当前下载流程

1. `server.py` 接收车号并将任务加入内存队列，最多同时运行两个任务。
2. `DownloadWorker` 查询标题和远程封面，然后调用 `jmcomic.download_album`。
3. 每张图片完成后上报进度；图片阶段最多上报到 94%。
4. 图片默认位于 `Pictures/<车号>/<章节>/`，随后合并为 `PDFs/<车号>.pdf`。
5. PDF 阶段上报 95%，完成后上报 100%。

## 已覆盖的行为

- 文件名和章节名按数字自然排序。
- PDF 按章节、页码顺序合并，并输出到指定目录。
- 下载器依次发送漫画信息、图片进度、PDF 阶段和完成事件。
- 下载异常通过错误回调上报。

## 后续阶段需要修正

- 当前预览使用远程封面 URL，不是本地下载的第一张漫画图片。
- 任务只保存在内存中，程序重启后不会恢复。
- PDF 生成会同时打开所有图片，大型漫画可能占用较多内存。
- 现有代码默认下载目录恰好为 `Pictures/<车号>`；重构时应由统一的路径规则保证。
